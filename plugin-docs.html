DeathClient JS Plugin API

v1.2.0  ·  EaglerCraft 1.8  ·  by Death68093

Write Minecraft mods in plain JavaScript — no Java, no build tools, no restarts. Paste your script or load a .js file and it runs live inside the game.

Overview

The DeathClient plugin system exposes a global object window.DeathClient (also accessible as just DeathClient) that lets JavaScript code interact with the game client at runtime. Every game tick (~50 ms) the Java side polls a set of queues your script writes to, and pushes fresh world state back into JavaScript.

The bridge is fully queue-based — you never call Java directly. You call a DeathClient method, it enqueues a command, and the game processes it on the next tick. This means:

  • All API calls are non-blocking and safe to call from any callback.
  • Actions have a maximum latency of one game tick (~50 ms).
  • The getPlayer() / getWorld() snapshots are always last tick's values, not real-time.

Console access: Open your browser's DevTools (F12) while the game is running. Type DeathClient.help() to print a quick API reference right in the console.

Loading Plugins

Method 1 — File Picker (recommended)

In the mod menu footer click [JS Plugins], then click Load Plugin (File). A native file dialog opens; select any .js file. Multiple files can be selected at once.

Method 2 — Paste

In the JS Plugins screen, click the + New tab. Type a name in the Plugin Name field, paste your source into the large text area, then click Load Pasted Source.

Method 3 — Browser Console

// Paste a one-liner mod straight from the console
DeathClient.registerMod({
  name: 'ConsoleMod',
  category: 'PLAYER',
  onUpdate: function() {
    var p = DeathClient.getPlayer();
    if (p && p.health < 5) DeathClient.chat('LOW HEALTH!');
  }
});

Hot-Reloading

In the Source tab of any loaded plugin, click Edit Source to open a full-screen browser text editor. Press Ctrl+S or click Save & Reload and the plugin is instantly unloaded and re-evaluated with your new source — no page refresh required.

Metadata Header

Place a comment block at the very top of your file. DeathClient reads @name, @description, and @version automatically and shows them in the plugin list.

// @name        My Awesome Plugin
// @description Does cool things in Minecraft
// @version     1.0
// @author      YourName

DeathClient.registerMod({ /* ... */ });

If @name is omitted, the plugin display name defaults to the filename (without extension). You can always rename it later in the plugin list.

Hello World

A minimal plugin that shows a chat message when enabled and prints your coordinates to the local chat every 5 seconds:

// @name     Coords Logger
// @version  1.0

var tickCount = 0;

DeathClient.registerMod({
  name:        'Coords Logger',
  description: 'Prints your coordinates every 5 seconds.',
  category:    'PLAYER',
  isCheat:     false,

  onEnable: function() {
    DeathClient.chat('§aCoords Logger enabled!');
    tickCount = 0;
  },

  onDisable: function() {
    DeathClient.chat('§cCoords Logger disabled.');
  },

  onUpdate: function() {
    tickCount++;
    if (tickCount % 100 !== 0) return; // ~5 seconds at 20 tps

    var p = DeathClient.getPlayer();
    if (!p) return;

    DeathClient.chat(
      '§7Pos: §f' + Math.floor(p.x) +
      ' / '       + Math.floor(p.y) +
      ' / '       + Math.floor(p.z)
    );
  }
});

Mod Lifecycle

A registered mod goes through the following states:

CallbackWhen it firesNotes
onEnableMod is toggled ONReset timers, clear state here
onDisableMod is toggled OFFClean up side-effects
onUpdateEvery game tick (~20/s) while enabledMain logic loop
onRender2DEvery rendered frame while enabledDraw HUD overlays. Can fire faster than 20/s
onKey(keyCode)Any key pressed while enabledUse keybinds set in mod settings
onChat(message)A chat message arrives while enabledmessage is the raw string

onRender2D is not a drawing context. You cannot draw pixels directly from JavaScript. Use it to read data and queue chat messages, or update variables that affect in-game rendering done by Java mods. True 2D drawing from JS is not currently supported.

Tick rate

onUpdate fires 20 times per second (once per Minecraft game tick). Use a counter to implement delays:

var timer = 0;

onUpdate: function() {
  timer++;
  if (timer < 40) return; // wait 2 seconds (40 ticks)
  timer = 0;
  // do thing every 2 seconds
}

Settings

Settings appear in the mod's settings screen inside the game. Declare them in the settings array when calling registerMod.

Setting types

typevalueExtra fieldsDescription
"boolean"true / false—Toggle switch
"number"numbermin, max, stepSlider
"mode"stringoptions: [...]Cycle through list
"string"string—Text field
 
DeathClient.registerMod({
  name: 'MyMod',
  settings: [
    { name: 'Enabled Feature', type: 'boolean', value: true  },
    { name: 'Speed',           type: 'number',  value: 1.5,
      min: 0.5, max: 5.0, step: 0.1 },
    { name: 'Mode',            type: 'mode',    value: 'Fast',
      options: ['Slow', 'Fast', 'Turbo'] },
    { name: 'Message',        type: 'string',  value: 'Hello!' },
  ],

  onUpdate: function() {
    // Read settings at runtime:
    var speed = DeathClient.getModSetting('MyMod', 'Speed');
    var mode  = DeathClient.getModSetting('MyMod', 'Mode');
    var msg   = DeathClient.getModSetting('MyMod', 'Message');

    if (mode === 'Turbo') {
      DeathClient.chat('§cTurbo mode! Speed = ' + speed);
    }
  }
});

Settings are stored in the JS object and persisted for the session. They are not saved to disk between page reloads — consider storing important values in localStorage from your plugin if needed.

Events

The event bus lets you react to things happening in the game from outside a registered mod, or share events across multiple mods in one plugin.

// Listen for any mod being enabled
DeathClient.on('mod_enable', function(data) {
  DeathClient.log('Mod enabled: ' + data.name);
});

// Listen for incoming chat
function onChat(data) {
  if (data.message.includes('!coords')) {
    var p = DeathClient.getPlayer();
    DeathClient.sendChat('My coords: ' + Math.floor(p.x) + ' ' + Math.floor(p.z));
  }
}
DeathClient.on('chat_receive', onChat);

// Remove the listener later
DeathClient.off('chat_receive', onChat);

Built-in event names

EventData payloadDescription
tickPlayer snapshot (or null)Fires every game tick
render2dnullFires every rendered frame
chat_receive{ message: string }Incoming chat message
mod_enable{ name: string }Any mod (Java or JS) enabled
mod_disable{ name: string }Any mod (Java or JS) disabled

World Data

World data is pushed from Java to JavaScript every tick. Always check for null before using it — data is unavailable while not in a world.

var p = DeathClient.getPlayer();
if (!p) return; // not in world yet

DeathClient.log('Health: ' + p.health + ' / ' + p.maxHealth);
DeathClient.log('On ground: ' + p.onGround);

var w = DeathClient.getWorld();
if (w && w.isRaining) DeathClient.chat('§9It is raining!');

var s = DeathClient.getServer();
DeathClient.log('Server: ' + (s ? s.ip : 'singleplayer'));

API Reference — Registration

registerMod (config: Object) → config Object

Registers a new mod and adds it to the mod list. The returned object is the same config reference — you can store it to read cfg._toggled later.

FieldTypeRequiredDescription
namestringYesUnique display name. Duplicate names are skipped.
descriptionstringNoShown in the tooltip and settings screen.
categorystringNoSee Categories. Default: "PLAYER".
isCheatbooleanNoIf true, mod is hidden and disabled in Legit Mode.
settingsArrayNoArray of setting objects. See Settings.
onEnablefunction()NoCalled when the mod is toggled on.
onDisablefunction()NoCalled when the mod is toggled off.
onUpdatefunction()NoCalled every tick while enabled.
onRender2Dfunction()NoCalled every frame while enabled.
onKeyfunction(keyCode)NoCalled on key press while enabled.
onChatfunction(message)NoCalled on incoming chat while enabled.
unregisterMod (name: string) → void

Removes a JS mod from the mod list at runtime. The mod is disabled first if it was active. Useful for self-unregistering after a one-time action.

API Reference — Chat & Commands

chat(message: string)→ void

Displays a message in the local chat window only. The server never sees it. Supports Minecraft colour codes (§a, §c, etc.).

sendChat(message: string)→ void

Sends a chat message to the server, exactly as if the player typed it. If the message starts with / it is treated as a command by the server.

sendCommand(command: string)→ void

Sends a slash command. The leading / is added automatically if missing. Equivalent to sendChat('/command').

API Reference — Mod Control

enableMod(name: string)→ void

Enables a mod by name. Works on both Java and JS mods. Legit Mode restrictions still apply.

disableMod(name: string)→ void

Disables a mod by name.

toggleMod(name: string)→ void

Toggles a mod's state — on if off, off if on.

isEnabled(name: string)→ boolean

Returns whether a mod is currently active. For JS mods this is synchronous (reads local state). For Java mods it reflects the last known state.

getMods()→ string[]

Returns an array of all registered mod names (both Java and JS).

getModInfo(name: string)→ Object | null

Returns a snapshot of a mod's metadata, or null if not found.

{ name, description, category, isCheat, enabled, settings: [...] }
setModSetting(modName: string, settingName: string, value: any)→ void

Changes a setting on any mod (Java or JS). The change is queued and applied on the next tick. value is coerced to a string internally, so pass numbers/booleans directly.

getModSetting(modName: string, settingName: string)→ any | null

Reads the current value of a setting from a JS mod only. Java mod settings are write-only from the JS side for now.

API Reference — World Data

getPlayer()→ PlayerState | null

Returns last tick's player snapshot. null when not in a world. See Player Object reference.

getWorld()→ WorldState | null

Returns last tick's world snapshot. See World Object reference.

getServer()→ ServerInfo | null

Returns current server connection info. See Server Object reference.

API Reference — Event Bus

on(event: string, callback: Function)→ void

Registers a listener for a named event. Multiple listeners per event are supported. See Events for the full list.

off(event: string, callback: Function)→ void

Removes a previously registered listener. You must pass the same function reference used in on() — anonymous functions cannot be removed, so store them in a variable first.

API Reference — Logging

log(message: string)

Prints to the browser console with [DC] prefix.

warn(message: string)

Prints a yellow warning to the browser console.

error(message: string)

Prints a red error to the browser console.

help()

Prints a compact API reference to the browser console.

Example — Custom HUD Line

Because direct 2D drawing from JS isn't supported, the cleanest way to add a HUD element is to create a mod that sends data to an existing Java HUD mod via a setting, or — more practically — display information via the chat() function triggered at intervals. Here is a pattern that announces your speed in chat periodically:

// @name Speed Announcer

var ticks = 0;
var lastX = 0, lastZ = 0;

DeathClient.registerMod({
  name:     'Speed Announcer',
  category: 'HUD',

  onUpdate: function() {
    var p = DeathClient.getPlayer();
    if (!p) return;

    ticks++;
    var dx = p.x - lastX, dz = p.z - lastZ;
    var speed = Math.sqrt(dx*dx + dz*dz); // blocks/tick
    lastX = p.x; lastZ = p.z;

    if (ticks % 20 === 0) { // every second
      DeathClient.chat('§7Speed: §e' + (speed * 20).toFixed(2) + '§7 b/s');
    }
  }
});

Example — Auto-Action (Low Health Flee)

Automatically runs a command and sends a warning when health drops below a threshold:

// @name     Health Guard
// @version  1.0

var alerted = false;

DeathClient.registerMod({
  name:    'Health Guard',
  category:'COMBAT',
  isCheat: false,
  settings:[
    { name: 'HP Threshold', type: 'number',
      value: 6, min: 1, max: 20, step: 1 },
    { name: 'Flee Command', type: 'string', value: '/hub' },
  ],

  onEnable:  function() { alerted = false; },

  onUpdate: function() {
    var p = DeathClient.getPlayer();
    if (!p) return;

    var threshold = DeathClient.getModSetting('Health Guard', 'HP Threshold');
    var cmd       = DeathClient.getModSetting('Health Guard', 'Flee Command');

    if (p.health <= threshold && !alerted) {
      alerted = true;
      DeathClient.chat('§c⚠ Low health! Running: ' + cmd);
      DeathClient.sendCommand(cmd);
    }
    if (p.health > threshold) alerted = false; // reset when healed
  }
});

Example — React to Chat

A command bot that responds to other players' messages:

// @name Chat Bot

var responses = {
  '!coords': function() {
    var p = DeathClient.getPlayer();
    return p ? Math.floor(p.x)+' / '+Math.floor(p.y)+' / '+Math.floor(p.z) : '?';
  },
  '!health': function() {
    var p = DeathClient.getPlayer();
    return p ? Math.floor(p.health) + ' HP' : '?';
  }
};

DeathClient.registerMod({
  name:    'Chat Bot',
  category:'PLAYER',

  onChat: function(message) {
    for (var trigger in responses) {
      if (message.indexOf(trigger) !== -1) {
        DeathClient.sendChat(responses[trigger]());
        return;
      }
    }
  }
});

Example — Rich Settings

A mod that demonstrates all four setting types together:

// @name Demo Settings

var mod = DeathClient.registerMod({
  name: 'Demo Settings',
  settings: [
    { name: 'Active',   type: 'boolean', value: true },
    { name: 'Strength', type: 'number',  value: 3.0,
      min: 1.0, max: 10.0, step: 0.5 },
    { name: 'Style',    type: 'mode',    value: 'Normal',
      options: ['Normal', 'Aggressive', 'Passive'] },
    { name: 'Label',    type: 'string',  value: 'Hello' },
  ],

  onUpdate: function() {
    var active   = DeathClient.getModSetting('Demo Settings', 'Active');
    if (!active) return;

    var strength = DeathClient.getModSetting('Demo Settings', 'Strength');
    var style    = DeathClient.getModSetting('Demo Settings', 'Style');
    var label    = DeathClient.getModSetting('Demo Settings', 'Label');

    DeathClient.log(label + ' | ' + style + ' | x' + strength);
  }
});

// You can also change settings programmatically:
DeathClient.setModSetting('Demo Settings', 'Strength', 7.5);

Example — Multi-Mod Plugin

One .js file can register as many mods as you like. They all appear in the mod list and are tracked under the same plugin entry:

// @name     Combat Suite
// @version  2.0
// @description A bundle of combat helpers.

/* ---- shared state ---- */
var playerCache = null;
DeathClient.on('tick', function(p) { playerCache = p; });

/* ---- Low HP Warning ---- */
DeathClient.registerMod({
  name: 'HP Warning', category: 'COMBAT', isCheat: false,
  settings: [{ name: 'Threshold', type: 'number', value:6, min:1, max:20, step:1 }],
  onUpdate: function() {
    if (!playerCache) return;
    var t = DeathClient.getModSetting('HP Warning', 'Threshold');
    if (playerCache.health <= t)
      DeathClient.chat('§c❤ LOW HEALTH: ' + Math.floor(playerCache.health));
  }
});

/* ---- Sprint Reminder ---- */
DeathClient.registerMod({
  name: 'Sprint Nag', category: 'MOVEMENT', isCheat: false,
  onUpdate: function() {
    if (playerCache && !playerCache.sprinting && playerCache.onGround)
      DeathClient.chat('§eReminder: press sprint!');
  }
});

Don't spam chat from onUpdate. The example above would print every tick. Add a tick counter or cooldown, as shown in earlier examples.

Limits & Gotchas

What you CAN do from JS

  • Read player position, health, motion, food, dimension, gamemode, XP every tick
  • Read world time, weather, difficulty
  • Read server IP and ping
  • Send chat messages and slash commands to the server
  • Display local-only chat notifications
  • Enable, disable, or toggle any mod by name
  • Change settings on any mod (Java or JS)
  • React to incoming chat messages via onChat
  • Register and unregister mods at runtime
  • Use the full browser JavaScript environment: fetch, localStorage, setTimeout, DOM APIs, etc.

What you CANNOT do from JS

  • Draw pixels directly on screen (no canvas/2D drawing context from JS)
  • Access the world's block data (no getBlock(x,y,z))
  • Access entity lists or read other players' data
  • Call Java methods directly (the queue bridge is the only channel)
  • Catch onRender3D events in JS

Performance

  • onUpdate fires 20 times per second. Keep logic lightweight — avoid nested loops or expensive string operations on every tick.
  • onRender2D can fire at 60+ times per second. Never run heavy logic there; use it only to read pre-computed values.
  • Unregister listeners with dc.off() when no longer needed, especially inside mods that disable frequently.
  • Large setTimeout or setInterval timers work fine but remember the page context is the EaglerCraft tab — they stop when the tab is hidden in most browsers.
